Routing Guide

Which Helper Should I Use?

Pick the narrowest documented helper that already fits the workflow. If none fits cleanly and the use case is repeated, submit a proposal instead of mutating the shared helper contract ad hoc from consuming app code.

Need Use Why Demo
Read-only message uiAlert(...) Fast semantic dialog without rebuilding modal actions. Alert Dialog
Confirm / cancel decision uiConfirm(...) Shared confirm semantics, variants, icons, and optional speech. Confirm Dialog
Prompt for one input uiPrompt(...) Short value collection without building a full modal form. Prompt Dialog
Free-form custom modal createModal(...) General shell for body content that does not fit a stricter wrapper. Modal
Known footer-action modal createActionModal(...) Shared action-footer shell with auto-busy state and declarative actions. Action Modal
Short custom modal form createFormModal(...) Strict row model, field errors, form error, and helper-owned busy submit flow. Form Modal
Shared login flow createLoginFormModal(...) Consistent structure with field-name remapping for backend compatibility. Login Preset
Session re-auth createReauthFormModal(...) Locked identifier and shared re-auth wording/behavior. Re-auth Preset
Standalone password entry createPasswordField(...) Shared password input with helper-owned show/hide toggle for non-modal forms or direct composition. Password Field
Status change workflow createStatusUpdateFormModal(...) Shared structure with app-supplied status vocabulary. Status Preset
Categorized reason-required flow createReasonFormModal(...) Reason select plus details textarea for monitoring and audit use cases. Reason Preset
Transient status feedback createToastStack(...) Shared semantic icons, optional speech, and auto-dismiss behavior. UI Utilities
Flat tabular data createGrid(...) Sorting, query state, pagination, virtualization, and dense action cells. Grid
Date selection createDatepicker(...) Single or range date capture with optional time input. Datepicker
File intake / upload queue createFileUploader(...) Queue-based uploads with retry and optional chunking. File Uploader
Hierarchical data only createTree(...) Expandable/selectable tree without forcing aligned non-tree columns. Tree
Hierarchical data with columns createTreeGrid(...) Hierarchy plus columns, tree-aware search, and lazy loading. Tree Grid
Hierarchy shown visually createHierarchyMap(...) Hierarchy-first visual map with overlay relationships and pan/zoom. Hierarchy Map
Standalone media viewing createMediaViewer(...) Dedicated image/video viewer with fit and navigation behaviors. Media Viewer
Rule

Proposal-First Reminder

  • If the helper library is close but missing a repeated capability, do not patch the shared contract ad hoc from app work.
  • Use the existing helper where it fits, and submit a proposal when the missing capability is repeated enough to belong in the shared library.
  • The routing guide is intentionally conservative. It is better to choose a lower-level helper explicitly than to overload a narrow preset.
