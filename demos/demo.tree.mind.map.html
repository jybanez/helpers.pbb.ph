Reference

Tree Mind Map

Canvas tree visualization for large rooted datasets. It is optimized for registry-style navigation where a separate app component owns selected-node details.

Overview Constructor Options Node Contract Events Methods

Use This When

  • You need a visual tree for hundreds or thousands of hierarchical nodes.
  • You need collapse/expand, pan, zoom, and multi-selection in one reusable surface.
  • The selected nodes should feed a separate detail panel, form, table, or action workflow.

Do Not Use This When

  • The user needs aligned columns, sorting, or report-style scanning; use createTreeGrid(...).
  • The main view is geographic location; use map tooling instead.
  • The data is a free-form graph with many arbitrary cross-links; use a graph-specific component.

Description

createTreeMindMap(...) renders nodes and edges on a Canvas surface for better large-data performance than a DOM-heavy SVG tree. The component keeps the data contract generic; apps adapt their own records into id, parentId, label, subtitle, status, and data.
