Hierarchy Map

Visual hierarchy explorer with a primary rooted structure and optional overlay links. It is hierarchy-first, not a free-form graph editor and not a geographic map.

Overview Constructor Options Properties Events Methods

Use This When

  • You need to show one rooted hierarchy with optional external entities and overlay links.
  • The viewer needs search, expand/collapse, zoom/pan, and node/link selection in one place.
  • You need a visual relationship map, not a purely tabular hierarchy.

Do Not Use This When

  • The primary need is tabular reporting or aligned tabular columns; use createTreeGrid(...).
  • The primary mental model is geography; use map tooling such as MapLibre instead.
  • You need a free-form graph editor with arbitrary multi-parent layout behavior.

Description

createHierarchyMap(...) keeps one primary tree and treats secondary links as overlays. In this demo, barangays keep their administrative parent chain while synthetic foundations connect in a separate external lane. That separation is the core design rule that keeps the layout readable.

createHierarchyMap(...) does not expose stable public instance properties as part of its contract. Read state through methods such as getState() and mutate behavior through helper methods like update(...), setSearchTerm(...), and selectNode(...).
