Login

Baseline helper config with required fields, busy submit, and field-error handling.

single-column field errors busy submit 
const formModal = createFormModal({
  title: "Operator Login",
  submitLabel: "Login",
  rows: [
    [{ type: "input", input: "email", name: "email", required: true }],
    [{ type: "input", input: "password", name: "password", required: true }],
  ],
  async onSubmit(values, ctx) {
    if (values.password !== "correct-password") {
      ctx.setErrors({ password: "Invalid password." });
      return false;
    }
    return true;
  },
});

Reference

Schema Form Modal

Shared helper for short operational modal forms. It composes over the existing modal shell and standardizes structure, validation, busy state, and submit handling.

Overview Constructor Options Properties Events Methods

Use This When

  • Short modal-bound workflows are enough.
  • You need field errors, form errors, and busy submit handling owned by the helper.
  • The flow can be expressed as a narrow row-based schema instead of app-local modal markup.

Do Not Use This When

  • The flow is a large CRUD editor with many fields and sections.
  • You need a multi-step wizard, page-level validation summary, or long-form editing workflow.
  • The layout depends on arbitrary grids, nested groups, or app-local modal body markup.

Description

createFormModal(...) is the shared direction for modal forms in PBB projects. It keeps the layout intentionally narrow: one or two items per row, helper-owned field rendering, and a stable submit lifecycle. Presets like login, re-auth, status update, and reason-required compose over it instead of bypassing it. Hosted helper fields such as ui.select and ui.treeSelect stay within the same contract instead of introducing app-local picker systems.

createFormModal(options) options: form-modal configuration object Instance with open/close, value, error, busy, and update methods

Options

OptionDefaultDescription
title""Modal title.
mode"create"Lightweight form mode surfaced to rules and state. Allowed values: create, edit.
contextnullNarrow top strip for acceptance-proof flows such as geodata hub add.
rows[]Row-based field schema. One or two items per row; hidden-only rows do not render visible gaps.
initialValues{}Initial payload values, including hidden fields.
visibleWhennullField-level conditional visibility rule. String values mean exact match, arrays mean inclusion, and all keys must match.
extraActions[]Additive footer actions rendered before helper-owned Cancel and Submit. Reserved IDs: cancel, submit.
extraActionsPlacement"end"Placement for additive footer actions. Allowed values: start, end. Use start to split extra actions away from helper-owned Cancel and Submit.
submitLabel"Submit"Primary action label.
busyMessage"Working..."Busy overlay message while async submit is pending.
onSubmit(values, ctx)nullSubmit hook. Return truthy to close, falsy to keep the modal open.
onClose(meta)nullClose callback with helper/meta context.

Properties

createFormModal(...) does not expose stable public instance properties as part of its working contract. Use methods such as getState(), getValues(), and isBusy() instead of reading instance fields directly.
